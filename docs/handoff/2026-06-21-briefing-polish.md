## 変更手順（項目別）

### 1. 資産総額・投資用キャッシュの初期マスク状態を「既定マスク」に揃える（恒久）
- 現状の不整合: `.maskable` は `¥***,***,***` を出荷するが、初期化IIFEが `body.masked` もアイコンも設定せず、`eye-open`（開いた目＝表示）が見えたまま → 「アイコンは表示・数字は非表示」のちぐはぐ。さらに初回タップで eye-closed に変わりつつ実数が**出る**逆転動作。
- 修正（生成HTMLの初期化IIFE末尾、`dataset.raw` 設定後）:
  ```js
  document.body.classList.add('masked');
  document.getElementById('eye-open').style.display = 'none';
  document.getElementById('eye-closed').style.display = '';
  ```
  → 既定で「目を閉じたアイコン＋`***`」。タップで実数表示＋開いた目に正しく反転。
- アプリ整合: `src/state.js` `statsMasked = localStorage('hm-stats-masked')!=='0'`（既定マスク）、`src/app.js` の eye-open=非マスク/eye-slash=マスク と一致。
- 対象は `.maskable` の3項目（資産総額・運用資産・投資用キャッシュ）一律。**キャッシュ比率（%）は `.maskable` を付けない**（常時表示・アプリ `render.js` と同じ）。
- spec §1 に「金額3項目は既定マスク・比率は常時表示」を1行明記。

### 2. 投資用キャッシュの `***,***,***`（9桁マスク）
- 項目1と同根（同じ `.maskable`）。**項目1の修正で連動解消**。個別変更なし。値は `dataset.raw` の実額（例 `¥52,430,523`）がタップで出る。

### 3. 「今週のまとめ」の「Valueタブへ」行を削除（恒久・spec修正必須）
- 現号 line 138 の `sum-row`「Valueタブへ … Valueタブでご確認ください」は、フッター line 223 の導線と重複。
- しかし**spec が §1 末尾の導線を必須化している**（spec §1 の「末尾に『保有の診断・売買判断は Value タブ』の一文を置く」＋チェックリスト該当行）。HTMLだけ消すと次号で復活する。
- 修正: **spec から §1-まとめの導線要求を削除**（フッター導線は残す＝唯一の導線に集約）。チェックリストの該当項目も削除。

### 4. セクション見出しクラスの統一（恒久）
- 現状 `.sum-h`（今週のまとめ・カード見出し）／`.sec-title`（番号バッジ付きセクション見出し）／`.subh`（カード内サブ見出し）が混在し、「今週のまとめ」が後続セクションと不整合に見える。
- ルール確定（spec に明記）:
  - **セクション見出し＝`.sec-title <b>N</b> 名前`**（番号バッジ付き）だけに限定。
  - **カード内サブ見出し＝`.subh` に統一**。`.sum-h` は廃止。
- 修正: 「今週のまとめ」を `<div class="sum-h">` → `<div class="subh">` に（トレンドの「今週」「来週の注目イベント」と同クラスに揃う）。spec のセクション順（現行版＝1サマリ/2トレンド/3マクロ/4メンター）に沿って番号バッジは `.sec-title` のみ。

### 5. 「来週の注目イベント」だけ文字が大きい（恒久・CSS）
- 原因: 見出し自体は `.subh`(11.5px) で同じ。**イベント本文 `.ev > div` がクラス無しで body 既定 16px を継承**し、周囲（トレンド表12px・主役/重し12.5px）より大きく見える。
- 修正: spec の CSS トークンに `.ev` 本文の font-size を追加。
  ```css
  .ev > div:last-child { font-size: 12.5px; }
  /* 必要なら .ev .d も 12px に正規化 */
  ```

### 6. メンター共通解の主張重複を排除（恒久・生成品質）
- 現号 line 217 `.insight` が「ハード/インフラに堀あり・ソフトに堀なし」の結論を二度書きし、各メンター主張（line 209-216 の `.ment` カード）を `.insight` 内で再列挙。さらに §1「今週のまとめ」のメンター共通解行とも二重。
- 修正（spec §4 ＋ SKILL に dedup 規則を追加）:
  - `.insight` メンター共通解は**核となる結論を一度だけ**述べる（同主張の二重記述禁止）。
  - 各メンターの個別主張は `.ment` で既出 → `.insight` では**再列挙しない**。4者が一致する一点＋テーマ配分への含意（1〜2文）に絞る。
  - §1「今週のまとめ」の共通解行（1行サマリ）と §4 `.insight`（配分インプリ）で**同じ要約を二度書かない**＝役割分担。
