# Wealth ネットワース：データモデル＆計算ロジック 正本（2026-07-21 確定）

Wealth タブの「総資産／運用資産／純資産」等の**数値定義の唯一の正本**。表示（見た目）は本 spec と別に設計する（数値だけ本 spec が決める）。ユーザー確定済み（2026-07-21 壁打ち）。

## 用語の前提（MF の 12 資産）
MoneyForward の BS カテゴリ＝12種。これが生データの器：
```
1 現金・預金  2 株式(現物)  3 株式(信用)  4 投資信託  5 債券  6 FX
7 暗号資産   8 保険       9 年金       10 ポイント  11 不動産(MF評価)  12 その他
MF総資産 = Σ(12資産)
```
別データ：
- **負債**（物件別ローン：自宅・収益）… `totals.liabilitiesTotal`（#589 で Worker/KV）
- **現実不動産評価**（保守 haircut）… `data/real-assets/*.json` の採用値 = `valueHowMa × haircut` = `totals.realAssetsTotal`

## 派生（決定論・ハルシネーション禁止）
```
生活資金        = ¥20,000,000（固定・生活防衛資金。現金から差し引く）

総資産（グロス） = MF総資産（= totals.mfNetWorth・MFそのまま。不動産はMF評価のまま盛り込み）

運用資産        = (現金・預金 − 生活資金)
                 + 株式(現物) + 株式(信用) + 投資信託 + 債券 + FX + 暗号資産 + 保険 + 年金
                 ★除外：ポイント・その他・不動産
                 （＝金融資産 − 生活資金。金融資産 = MF総資産 − 不動産 − その他 − ポイント）

不動産補正      = 不動産(MF評価) − 現実不動産評価(realAssetsTotal)   ← MFの過大評価の戻し

純資産          = MF総資産 − 不動産補正 − 負債
               （= 金融 + ポイント + その他 + 現実不動産 − 負債。実質の手取り）
```

### ウォーターフォール（表示の骨・自覚できる並び）
```
MF総資産（グロス） −[不動産の盛り分] −[負債] ＝ 純資産（手取り）
```
補助：`運用資産 = MF総資産 −（ポイント + その他 + 不動産 + 生活資金）` の透明化行。

## 決定の根拠（ユーザー確定・2026-07-21）
- 運用資産：**年金・保険・暗号は含む／ポイントは含まない／その他・不動産は含まない**。生活資金¥20M固定・現金から控除。
- 総資産：**MF総資産そのまま**（不動産もMF評価）。
- 純資産：**不動産を現実値（保守haircut）に戻し、負債を控除**（＝Mulmo/アプリの付加価値）。
- 信用建玉：モデル化しない（今後も使わない・現状ゼロ）。

## データ源の現況（実装の前提）
| 必要値 | 取得元 | 状態 |
|---|---|---|
| MF総資産 | `totals.mfNetWorth` | ✅ 取得済 |
| 現金/株/投信/債券/FX/暗号 | holdings（Σ=imported） | ✅ 取得済 |
| **年金・保険（カテゴリ値）** | `mf-history.json` の pension/insurance 列 | ✅ 取得済（保有スクレイプは除外だがBS内訳にあり） |
| **不動産(MF評価)** | MF `/bs/portfolio` 不動産テーブル（config `realEstate`・#580） | ⚠️ 設定あり・**実機未確認**＝要実装/確認 |
| その他 | MF BS「その他・事業性資産」 | ⚠️ 現状カテゴリ除外・**要捕捉**（純資産計算に必要） |
| 現実不動産（haircut） | `data/real-assets/*.json`（realAssetsTotal） | ✅ 取得済 |
| 負債 | `totals.liabilitiesTotal`（Worker/KV・#589） | ✅ 取得済 |

→ **不足は実質「不動産(MF評価)」と「その他」の捕捉だけ**。他は既存 totals ＋ mf-history で揃う。

## 実装分解
- **P1 パイプライン**：fetch が「不動産(MF評価)」「その他」を捕捉し、canonical に MF12資産フル＋不動産(MF)＋その他 を出す。
- **P2 計算**：`networth.js` の集計を本 spec の派生式に差し替え（`netWorthComputed` の旧式 `imported + realAssetsTotal − liabilitiesTotal` は**廃止**、`純資産 = mfNetWorth − 不動産補正 − 負債` に）。
- **P3 表示**：Wealth タブのネットワース面を上記数値＋ウォーターフォールで描く（**見た目は別途デザイン**）。

> ★旧 `netWorthComputed`（#577）は本 spec で**再定義**。load-bearing なので移行時に networth.js の消費側と揃えて変更する（下流の壊れ注意）。
