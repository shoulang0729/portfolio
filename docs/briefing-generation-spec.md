# Briefing 生成スペック（クラウド生成プロンプト貼り付け用）

> **用途**: 日次 Briefing HTML を生成するクラウド側ルーティン（MulmoClaude）のプロンプトに、この章を**そのまま貼る**ための確定仕様。（**2026-06-25 改名**：旧「週次」運用→現在は日次バッチで生成。スキルは `briefing`、ラベルに「週次」は出さない。）
> repo 側の `docs/investment-system-architecture.md` は内部メモで生成は駆動しない。**生成挙動を変えたいときはクラウドのプロンプトを直すこと。**
> **役割の住み分け（2026-06-19 改定）**: Briefing は「**世界と物語**（マクロ地合い・メンター見解・その時点のスタンス）」＝トップダウンの読み物に専念する。**保有の判断（ヘルス／集中度／アクション／PER採点・バリュエーション）はアプリの「Value」タブへ移設**したため、Briefing からは旧§4ヘルス・§5トリガー＆アクション・§7PER を**撤去**した（4セクション構成）。
>
> **視覚リファレンス（2026-06-25 v4新版）＝`docs/handoff/assets/2026-06-25-briefing-layout-reference.html`**。見出しアイコン化・番号バッジ/nav 廃止・§1資産2行・TOPIX/ハンセン%埋め を反映済み。構造・class・CSS はここから踏襲する（移行期は過去号HTMLを参照しない）。

---

## 0. 出力の基本

- **自己完結のモバイル HTML 1ファイル**。**生成日時でバージョン管理**し `data/briefings/YYYY-MM-DD-HHMM.html`（CST/UTC+8）に保存。**既存号は上書きしない＝毎回新規ファイル**（ニュースも仕様も変わり続けるので全版を履歴として残す。日次でも上書きしない）。
  - 版IDの取得: bash `TZ='Asia/Shanghai' date +%Y-%m-%d-%H%M`（ファイル名用）/ `TZ='Asia/Shanghai' date +'%Y-%m-%d %H:%M'`（index の date/title 用）。
  - `index.json` の `issues` 配列の**先頭に新版を追加**（過去号は消さない）。`updated` も版IDに更新。
    - `date` = `"YYYY-MM-DD HH:MM"`（**固定幅で辞書順＝新しい順にソート可能**。アプリ `src/briefing.js` は date 降順で最新を表示し、過去号プルダウンに全版を出す＝**コード変更不要**）。
    - `title` = `"Briefing YYYY/MM/DD HH:MM"`（同日複数版を時刻で区別）。
    - `path` = `"data/briefings/YYYY-MM-DD-HHMM.html"`。
- アプリ配色トークンを内蔵（`--bg/--surface/--surface2/--border/--text/--text2/--accent/--up/--down/--stance`）。`:root` ＋ `@media (prefers-color-scheme:dark)` ＋ `html[data-theme="light"|"dark"]` の3系統を定義。
- 色は**日本市場慣例**：`--up`＝赤（上昇/含み益）、`--down`＝緑（下落/含み損）。
- **見出しクラス規約（2026-06-25 改定）**: セクション見出しは**カード内の先頭**に置く `.card-ttl`＝**SVGアイコン箱＋セクション名のみ**（番号バッジ・番号テキストは出さない）。Riskタブと同作法＝自前SVGスプライト（`<symbol>`＋`<use>`・stroke 1.7）、アイコン箱は28px角の `.tic`（背景 `--surface2`・色 `--accent`）。**旧 sticky な `.sec-title`（番号バッジ `<b>N</b> 名前`）は廃止**。アイコン対応＝サマリ:リスト/トレンド:波形(pulse)/マクロ:地球(globe)/メンター:人(users)。カード内サブ見出しは **`.subh` に統一**（`.sum-h` は使わない）。混在させない。
- **本文サイズの正規化**: イベント本文がクラス無しで body 既定16pxを継承して周囲より大きく見えるのを防ぐ。`.ev > div:last-child { font-size: 12.5px; }`（必要なら `.ev .d` も 12px に正規化）＝トレンド表(12px)・主役/重し(12.5px)と同等に。
- `index.json` 例（先頭が最新版・過去号を蓄積）:
  ```json
  { "updated": "2026-06-14 23:45",
    "issues": [
      { "date": "2026-06-14 23:45", "title": "Briefing 2026/06/14 23:45", "path": "data/briefings/2026-06-14-2345.html" },
      { "date": "2026-06-14 09:30", "title": "Briefing 2026/06/14 09:30", "path": "data/briefings/2026-06-14-0930.html" }
    ] }
  ```

## 1. ヘッダー（`<header>`）

- **大見出し `<h1>Briefing</h1>` は出さない**（アプリのタブで明示済み・重複）。
- `.date` は**日付＋生成時刻**：`YYYY/MM/DD（曜）HH:MM`。**「週次」等のカデンス語は出さない**（日次運用のため）。**スタンスはここに出さない**（→ §2 トレンド先頭へ）。
- **ジャンプ用 `.nav`（ピル）は廃止（2026-06-25）**。ヘッダーは「日付（左）＋今すぐ生成（右）」だけ。セクション名はアイコンで各カード内に表示するので、ヘッダーのナビは出さない。

## 2. セクション構成（順序・番号・nav を常に一致）

| # | id | 見出し |
|---|-----|--------|
| 1 | s1 | サマリ |
| 2 | s2 | トレンド |
| 3 | s3 | マクロ |
| 4 | s4 | メンター |

> **撤去済み（Value タブへ移設・2026-06-19）**: 旧 ヘルスチェック／トリガー＆アクション／PERアナリシス は**生成しない**。Briefing は保有の個別判断（集中度・売買発議・銘柄バリュエーション）を扱わない。それらは常時ライブの「Value」タブの領分。

## §1 サマリ
- **1カードのみ**。`.card-ttl`（リストアイコン＋「サマリ」）＋資産2行構成。**「今週のまとめ」カードは廃止（2026-06-25）**＝地合い→§2／メンター共通解→§4／来週の注目→§2 と重複していたため。スタンスも§1には出さない（§2先頭のみ）。
- **資産は2行（3項目→2行に圧縮）**：
  - 1行目：**資産総額**（大・`.s1-big`）と**運用資産**（中・`.s1-mid`）を縦線区切りで**横並び**（`.s1-grid` の2セル）。各セルは小ラベル（`.s1-lbl`・**左寄せ＝金額の左上**）＋金額。**運用資産の金額は右寄せ**（`.s1-mid{text-align:right}`）、資産総額は左寄せ。
    - **崩れ対策（2026-06-26・必須）**：9桁の金額が横並びでスマホ幅からはみ出す事故を防ぐ。**`overflow:hidden` で隠さない**（金額を見切れさせない）。代わりに**コンテナクエリでセル幅にフォント追従**＝`.s1-cell{container-type:inline-size}` ＋ `.s1-big{font-size:clamp(15px,12.5cqi,23px)}`（運用資産 `clamp(13px,10.5cqi,18px)`）＋`white-space:nowrap`＋`tabular-nums`。`vw` は使わない（セルが画面半分なのに全画面幅基準で計算され見切れる）。`.s1-cell .maskable{display:block}` でマスク用 inline-block を上書き。中央の縦線は左右に余白（`.s1-cell + .s1-cell{margin-left:13px;padding-left:13px;border-left:…}`）。
  - 2行目：**投資用キャッシュ**（`.s1-cash`）＝ラベル＋比率pill（`比率 NN.N%`）＋右寄せ金額。
- 資産総額・運用資産・投資用キャッシュは**1円単位（カンマ区切り）**。キャッシュ比率は **投資用キャッシュ ÷ 運用資産**。
- 金額の目隠しは blur やバーではなく、数字のみを `*` に置換して `¥***,***,***` 形式。
- **金額3項目（資産総額・運用資産・投資用キャッシュ）は既定マスク**（アプリ本体と同じ「目を閉じるアイコン＋`***`」）。タップで実数（`dataset.raw` の1円単位）表示＋アイコン反転。**キャッシュ比率（%）は `.maskable` を付けず常時表示**。**目アイコン（マスク切替ボタン）は §1 の `.card-ttl` 右端**（`margin-left:auto`）に置く。
  - 初期化 IIFE 末尾（`dataset.raw` 設定後）に必ず既定マスクを確定させる:
    ```js
    document.body.classList.add('masked');
    document.getElementById('eye-open').style.display = 'none';
    document.getElementById('eye-closed').style.display = '';
    ```
- **鮮度バナー**（`pack.holdings.stale===true` のとき）は資産2行の直下に控えめに（§段C手順を参照）。

## §2 トレンド
- 1枚のカードに「今週」と「来週」をまとめる。
- **スタンスは黄色の一言だけ（2026-06-25）**：`.trend-stance` 内は `<span class="stance">⚠ 中立〜やや強気（…）</span>` の**一行のみ**。**旧版の後続の長い解説文（「— 構造は…」以降）は出さない**（理由は地合い表・追い風/向かい風で十分に展開されるため）。位置は §2 `今週` 直後（ヘッダーには出さない）。
- 今週の地合い表、追い風/向かい風、来週のイベントをこの順に置く。

## §3 マクロ
- 指標テーブル（`table.mkt`）：日経平均・**TOPIX**・ダウ・S&P500・Nasdaq総合・**ハンセン**・米10年債・**ドル円**・**VIX**・米CPI。列＝1週/1月/3月/6月。
- **TOPIX・ハンセンは必ず%を埋める（2026-06-25 改定）**＝旧版の空欄「—」（蓄積待ち）を廃止。データ源＝`market-snapshot.mjs` が **ハンセン=Yahoo `^HSI`（水準＋%）／TOPIX=Yahoo 上場ETF `1308.T` の調整後終値から%算出**（実指数がYahoo/FREDに無いためETF代理）。TOPIXの `.lv` 水準は `displayValue`（実指数値・任意の1検索）があればそれ、無ければ ETF価格＋「1308.T換算」注記。
- **ドル円・VIX は各期間セルを「実数値（指数水準/レート）」で表示**（％ではない）。セルの色（`up`/`down`）は**現在値への騰落方向**。
- 株価指数は騰落%（日本式＝赤=上昇/緑=下落）、債券は bp。`.lv` に現在値（例「現在 160.6」）。
- 末尾 `.note` に概算である旨と、TOPIX/ハンセンが Yahoo（1308.T調整後／^HSI）算出である旨。

## §4 メンター
- `.ment`（who＋claim）：じっちゃま（広瀬隆雄）／中島聡／ゆな先生／藤沢数希／後藤達也。後藤は**中立・数字ドリブンの決算/マクロ分解**レンズ（売買callは出さない＝構造理解の補完。例：キオクシア=利益率/PER/LTAの分解、AIと物価=メモリ高騰→利上げ観測）。
- `.insight` で5者の共通解＋それが**テーマ配分に示す方向**（※個別銘柄の売買発議はしない＝Value タブ）。
- **重複排除（生成品質）**: `.insight` は核となる結論を**一度だけ**述べる（同主張の二重記述禁止）。各メンターの個別主張は `.ment` で既出のため `.insight` で**再列挙しない**。5者が一致する一点＋配分への含意（1〜2文）に絞る。
- **`.insight` 末尾の「※保有の診断・売買発議・PER採点は Value タブをご覧ください」は出さない（2026-06-25）**＝フッターに同じ Value 導線があり重複のため。Value 導線はフッター1箇所のみ。
