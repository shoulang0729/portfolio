# Briefing 生成スペック（クラウド生成プロンプト貼り付け用）

> **用途**: 週次 Briefing HTML を生成するクラウド側ルーティン（MulmoClaude）のプロンプトに、この章を**そのまま貼る**ための確定仕様。
> repo 側の `docs/investment-system-architecture.md` は内部メモで生成は駆動しない。**生成挙動を変えたいときはクラウドのプロンプトを直すこと。**
> **役割の住み分け（2026-06-19 改定）**: Briefing は「**世界と物語**（マクロ地合い・メンター見解・週次のスタンス）」＝トップダウンの読み物に専念する。**保有の判断（ヘルス／集中度／アクション／PER採点・バリュエーション）はアプリの「Value」タブへ移設**したため、Briefing からは旧§4ヘルス・§5トリガー＆アクション・§7PER を**撤去**した（4セクション構成）。

---

## 0. 出力の基本

- **自己完結のモバイル HTML 1ファイル**。**生成日時でバージョン管理**し `data/briefings/YYYY-MM-DD-HHMM.html`（CST/UTC+8）に保存。**既存号は上書きしない＝毎回新規ファイル**（ニュースも仕様も変わり続けるので全版を履歴として残す。週次でも上書きしない）。
  - 版IDの取得: bash `TZ='Asia/Shanghai' date +%Y-%m-%d-%H%M`（ファイル名用）/ `TZ='Asia/Shanghai' date +'%Y-%m-%d %H:%M'`（index の date/title 用）。
  - `index.json` の `issues` 配列の**先頭に新版を追加**（過去号は消さない）。`updated` も版IDに更新。
    - `date` = `"YYYY-MM-DD HH:MM"`（**固定幅で辞書順＝新しい順にソート可能**。アプリ `src/briefing.js` は date 降順で最新を表示し、過去号プルダウンに全版を出す＝**コード変更不要**）。
    - `title` = `"Briefing YYYY/MM/DD HH:MM"`（同日複数版を時刻で区別）。
    - `path` = `"data/briefings/YYYY-MM-DD-HHMM.html"`。
- アプリ配色トークンを内蔵（`--bg/--surface/--surface2/--border/--text/--text2/--accent/--up/--down/--stance`）。`:root` ＋ `@media (prefers-color-scheme:dark)` ＋ `html[data-theme="light"|"dark"]` の3系統を定義。
- 色は**日本市場慣例**：`--up`＝赤（上昇/含み益）、`--down`＝緑（下落/含み損）。
- **見出しクラス規約**: セクション見出しは番号バッジ付き `.sec-title`（`<b>N</b> 名前`）**のみ**。カード内サブ見出しは **`.subh` に統一**（`.sum-h` は使わない＝「今週のまとめ」も `.subh`）。混在させない。
- **本文サイズの正規化**: イベント本文がクラス無しで body 既定16pxを継承して周囲より大きく見えるのを防ぐ。`.ev > div:last-child { font-size: 12.5px; }`（必要なら `.ev .d` も 12px に正規化）＝トレンド表(12px)・主役/重し(12.5px)と同等に。
- `index.json` 例（先頭が最新版・過去号を蓄積）:
  ```json
  { "updated": "2026-06-14 23:45",
    "issues": [
      { "date": "2026-06-14 23:45", "title": "Briefing 2026/06/14 23:45", "path": "data/briefings/2026-06-14-2345.html" },
      { "date": "2026-06-14 09:30", "title": "Briefing 2026/06/14 09:30", "path": "data/briefings/2026-06-14-0930.html" }
    ] }
  ```

## 1. ヘッダー（`<header>`）

- **大見出し `<h1>Briefing</h1>` は出さない**（アプリのタブで明示済み・重複）。
- `.date` は**日付のみ**：`YYYY/MM/DD（曜）週次`。**スタンスはここに出さない**（→ §2 トレンド先頭へ）。
- `.nav`（ピル）はセクション順と一致：`サマリ / トレンド / マクロ / メンター`。

## 2. セクション構成（順序・番号・nav を常に一致）

| # | id | 見出し |
|---|-----|--------|
| 1 | s1 | サマリ |
| 2 | s2 | トレンド |
| 3 | s3 | マクロ |
| 4 | s4 | メンター |

> **撤去済み（Value タブへ移設・2026-06-19）**: 旧 ヘルスチェック／トリガー＆アクション／PERアナリシス は**生成しない**。Briefing は保有の個別判断（集中度・売買発議・銘柄バリュエーション）を扱わない。それらは常時ライブの「Value」タブの領分。

---

## §1 サマリ
- 1枚目は資産総額カード。資産総額・運用資産・投資用キャッシュは**1円単位（カンマ区切り）**で表示。
- キャッシュ比率は **投資用キャッシュ ÷ 運用資産**。
- 金額の目隠しは blur やバーではなく、数字のみを `*` に置換して `¥***,***,***` 形式にする。
- **金額3項目（資産総額・運用資産・投資用キャッシュ）は既定マスク**（アプリ本体と同じ「目を閉じるアイコン＋`***`」）。タップで実数（`dataset.raw` の1円単位）表示＋アイコンが開いた目に反転。**キャッシュ比率（%）は `.maskable` を付けず常時表示**（アプリ `render.js` と同じ）。
  - 初期化 IIFE 末尾（`dataset.raw` 設定後）に必ず既定マスクを確定させる（これが無いと「開いた目＋`***`」のちぐはぐ＋初回タップ逆転になる）:
    ```js
    document.body.classList.add('masked');
    document.getElementById('eye-open').style.display = 'none';
    document.getElementById('eye-closed').style.display = '';
    ```
- 2枚目は `今週のまとめ`。**地合い・メンター共通解・来週の注目**を短く列挙する（※保有ヘルス／個別アクション／PER は扱わない）。**Value タブへの導線はここに置かない**（重複回避のため導線はフッター1箇所に集約）。

## §2 トレンド
- 1枚のカードに「今週」と「来週」をまとめる。
- スタンス（⚠ 黄信号 等）は §2 トレンドの `今週` 直後 `.trend-stance` に置く（ヘッダーには出さない）。
- 今週の地合い、追い風/向かい風、来週のイベントをこの順に置く。

---

## §3 マクロ
- 指標テーブル（`table.mkt`）：日経平均・TOPIX・ダウ・S&P500・Nasdaq100・ハンセン・米10年債・**ドル円**・**VIX**。列＝1週/1月/3月/6月/1年。
- **ドル円・VIX は各期間セルを「実数値（指数水準/レート）」で表示**（％ではない）。セルの色（`up`/`down`）は**現在値への騰落方向**。
- 株価指数は騰落%、債券は bp のまま。`.lv` に現在値（例「現在 160.6」）。
- 末尾 `.note` に概算である旨。

## §4 メンター
- `.ment`（who＋claim）：じっちゃま（広瀬隆雄）／中島聡／ゆな先生／藤沢数希。
- `.insight` で4者の共通解＋それが**テーマ配分に示す方向**（※個別銘柄の売買発議はしない＝Value タブ）。
- **重複排除（生成品質）**: `.insight` は核となる結論を**一度だけ**述べる（同主張の二重記述禁止）。各メンターの個別主張は `.ment` で既出のため `.insight` で**再列挙しない**。4者が一致する一点＋配分への含意（1〜2文）に絞る。§1「今週のまとめ」の共通解行（1行サマリ）と §4 `.insight`（配分インプリ）で**同じ要約を二度書かない**（役割分担）。

## アプリ側クローム（参考・repo で管理済み／生成HTMLとは別）
クラウド生成は §1〜§4 の HTML だけを作る。下記はアプリの `src/briefing.js`＋CSS 側で確定済み（生成側は触らない）：
- 上部ツールバー：**号タイトル/日付ラベルは出さない**。白い背景バンドなし。生成ボタンは塗りでなく**ゴースト＋細線 SVG アイコン**（絵文字 💬 は使わない）。
- 最新号は iframe 表示、過去号は下にリンク一覧（`index.json` に蓄積されれば自動表示）。
